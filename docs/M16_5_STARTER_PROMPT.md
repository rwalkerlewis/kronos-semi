### Phase B: psi Dirichlet at the Schottky facet

The Schottky contact's psi value is the metal Fermi level
relative to the semiconductor intrinsic level, not the
majority-side equilibrium psi the ohmic builder computes.
Equilibrium and bias-sweep paths both need this.

1. Extend `ContactBC` in `semi/bcs.py`:
   - New field `barrier_height_eV: float | None = None`. Default
     `None` for non-Schottky contacts.
   - `resolve_contacts` reads it from the JSON entry and threads
     it onto the `ContactBC` instance for Schottky contacts;
     leaves `None` for everything else.
2. Replace the `NotImplementedError` at `build_psi_dirichlet_bcs`
   L190-193 with the Schottky psi expression. The metal Fermi
   level sits `barrier_height_eV` below the conduction band on
   the metal side; in the semiconductor's intrinsic-Fermi
   convention, the equilibrium psi at the Schottky facet is:
   ```
   psi_Schottky_eq = -(barrier_height_eV - V_t * ln(N_C / n_i)) / V_t
   ```
   (cite Sze § 3.4 for the derivation in the Phase B docstring;
   this expression assumes the Boltzmann limit, which the
   benchmark uses; under FD, the `V_t * ln(N_C / n_i)` term is
   replaced by the FD inversion of the bulk equilibrium n_eq
   value; defer the FD generalization to a follow-up if M16.5
   ships under Boltzmann statistics by default).
   ```
   psi_bc = psi_Schottky_eq + c.V_applied / sc.V0
   ```
   Apply as the Dirichlet on `V_psi`.
3. Extend `build_dd_dirichlet_bcs` to apply the same psi BC on
   Schottky facets. Unlike ohmic, **do not** apply Dirichlet
   BCs on phi_n / phi_p at Schottky facets; those rows are
   handled by the Robin surface integral added in Phase C.
   Update the L227 filter to include `"schottky"` and add a
   branch alongside the existing `gate` branch:
   ```python
   if c.kind == "schottky":
       psi_bc = _schottky_psi_eq(c, ref_mat, sc) + c.V_applied / sc.V0
       bcs.append(fem.dirichletbc(
           PETSc.ScalarType(psi_bc), dofs_psi, spaces.V_psi))
       continue
   ```
   (factor the equilibrium psi computation into a private helper
   `_schottky_psi_eq(contact, ref_mat, sc)` so the equilibrium
   and DD builders share the same code).

### Phase C: Robin thermionic-emission surface form on the continuity rows

The thermionic-emission BC is a Robin condition on
`J_n . n` and `J_p . n` at the Schottky facet:

```
J_n . n = q v_n_thermal (n - n_eq exp((V_a - phi_n) / V_t))
J_p . n = -q v_p_thermal (p - p_eq exp((phi_p - V_a) / V_t))
```

where `v_n_thermal = sqrt(kT / (2 pi m_n*))`, the electron
thermal velocity (Si: ~2.05e7 cm/s at 300 K), and `n_eq` is the
equilibrium n at the metal contact (closed-form from
`barrier_height_eV` and `N_C`). In Slotboom primary unknowns
(`phi_n` is primary), the Robin form integrates against the
test functions on the continuity rows. See Sze § 3.4 for the
sign convention; for the cathode-on-the-right Schottky diode
(electrons flow into the metal under forward bias), the Robin
form adds a positive boundary integral to the electron
continuity row.

1. Extend `semi/physics/drift_diffusion.py`:
   - Add a `schottky_facets: list[tuple[int, dict]]` (or a
     dict keyed by facet tag) parameter to
     `build_dd_block_residual` and `_mr`. Each entry carries the
     barrier_height_eV, the applied voltage, and the facet tag.
     Default empty list (no Schottky contacts; bit-identical to
     v0.20.0).
   - When the list is non-empty, build the Robin surface form
     using UFL `ds(facet_tag)` and add it to the electron and
     hole continuity rows. Cite Sze § 3.4 in the docstring.
   - The thermal velocities `v_n_thermal`, `v_p_thermal` are
     computed once at scaling time from kT and the effective
     masses (`m_n_star`, `m_p_star`) on `Scaling`. Add
     `m_n_star` and `m_p_star` to `Scaling` and to the Si entry
     in `semi/materials.py`. Si defaults: `m_n_star = 0.26 m_0`,
     `m_p_star = 0.39 m_0` (per Sze Table 1; effective masses
     for thermionic emission, not the conductivity effective
     masses). Other materials carry `None` placeholders.
2. Runner threading. `bias_sweep`, `transient`, `ac_sweep`, and
   `equilibrium` (the four runners that consume the DD form
   builder) extract the Schottky facets from the resolved contact
   list and pass them to the form builder:
   ```python
   schottky_facets = [
       (c.facet_tag, {"barrier_height_eV": c.barrier_height_eV,
                       "V_applied": c.V_applied})
       for c in contacts if c.kind == "schottky"
   ]
   F_list = build_dd_block_residual(..., schottky_facets=schottky_facets)
   ```
   The `mos_cv`, `mos_cap_ac`, and `resistor_3d` runners do not
   touch the Schottky facets but should still pass the empty
   list through (one-line addition).
3. Pure-Python tests in `tests/test_recombination.py` (or a new
   `tests/test_thermionic.py` if the thermionic helpers are
   factored out as standalone functions):
   - Closed-form `n_eq` from `barrier_height_eV` matches a
     manual calculation at three sample barrier heights.
   - Thermal-velocity computation matches the textbook value at
     T = 300 K within 1 %.
   - `n_eq` reduces to the textbook ohmic limit when the barrier
     height approaches zero.
4. FEM-side smoke test in `tests/fem/test_schottky_surface.py`:
   - A trivial 1D pn diode with one Schottky contact assembles
     without error.
   - The Schottky surface integral contributes a non-zero residual
     entry on the electron continuity row at the Schottky facet
