# M13.1 follow-up #5 — close-out blocker

Date: 2026-04-27
Branch: copilot/devm131-sg-time-loop-followup-5 (requested name was
`dev/m13.1-sg-time-loop-followup-5`; the branch name preset by the

## Outcome

All three hypotheses laid out in the issue were attempted exactly once
(per the "do not iterate within a hypothesis" cap). None closes the
SG-in-time-loop xfail with the required `rel_err < 1e-4`. Per the
escape-hatch instruction in the issue, no speculative fix has been
pushed and the production code is unchanged.

## Reproduction setup

- 1D pn junction, 100 cells, V_F = 0.3 V, tau = 1 ns, dt = 50 ps,
  BDF2, t_end = 30 ns, `use_sg_flux=True`.
- Pre-fix baseline (PR #45 / PR #46 state): SNES line-search
  divergence at step 52, `reason=-5` (`DIVERGED_LINE_SEARCH`),
  ~2.6 ns of simulated time. Confirmed verbatim in this session.
- Reference J_ss from `run_bias_sweep` at the same V_F: **+2.884 A/m^2**.

## Hypothesis A — PETSc `vinewtonrsls` VI solver

Implementation: replaced `solve_sg_block_1d` body to:
- set `snes_type=vinewtonrsls`
- build `xl`/`xu` PETSc Vecs of the block layout
- set `psi` block to `(NINFINITY, INFINITY)` and `n_hat`/`p_hat`
  blocks to `(1e-30, INFINITY)`
- call `snes.setVariableBounds(xl, xu)` before `solve()`
- removed the `setUpdate` orthant clip (VI replaces it)

Result: **FAIL at step 31, `reason=-6` (`DIVERGED_FUNCTION_COUNT`)**.
The VI active-set logic shows worse robustness than the Galerkin/clip
combination here — the active set churns long before it can stabilise,
exhausting the function-count budget. The 1e-30 floor is so far below
the relevant carrier scale (~1e-9 in n_hat units) that the constraint
is essentially never active in physical space, which suggests the
issue truly is not pure positivity loss.

## Hypothesis B — alternative SNES line searches

Tried each of `cp`, `l2`, `nleqerr`, `basic` exactly once, with all
other settings unchanged from the working PR #45/PR #46 path
(`snes_type=newtonls`, MUMPS LU).

| line search | result                                                  |
|-------------|---------------------------------------------------------|
| `cp`        | runs all 600 steps; **J_anode = -151.17 A/m^2** (rel_err = 53.4 vs J_ss = +2.884; opposite sign — converged to a non-physical critical point) |
| `l2`        | FAIL step 42, `reason=-9` (`DIVERGED_DTOL`)             |
| `nleqerr`   | FAIL step 38, `reason=-9`                               |
| `basic`     | FAIL step 37, `reason=-9` (full Newton step blows up)   |

`cp` "converges" because it accepts any step that reduces a
critical-point measure, and the (n,p)-form residual at large |dpsi|
admits multiple critical points along the search direction. The other
three either diverge earlier than `bt` or behave the same.

## Hypothesis C — direct SG assembly (no Galerkin subtraction)

Implementation:
- Patched `_build_transient_residual` to omit the convection-diffusion
  blocks from F_n / F_p (mass + SRH + history + Poisson only).
- Wrote a `_sg_jacobian_direct` helper that returns the SG-only
  per-edge Jacobian (the SG half of `assemble_sg_jacobian_correction_1d`,
  without subtracting the Galerkin CD piece).
- Replaced `solve_sg_block_1d` so the residual callback adds the SG
  flux directly via `assemble_sg_residual_1d` (no Galerkin
  subtraction) and the Jacobian callback adds `_sg_jacobian_direct`
  directly. Set `MAT_NEW_NONZERO_ALLOCATION_ERR=False` on the
  Jacobian matrix because the no-CD UFL form has tighter sparsity
  than the SG stencil requires.
- Kept the orthant clip and the M13 sign convention.

Result: **FAIL at step 52, `reason=-5` — same step, same reason as
the unmodified Galerkin-minus-Galerkin-plus-SG baseline.**

This is the strongest piece of numerical evidence in the session:
the cancellation pattern is **not** the source of the divergence.
At step 52 the residual at the failing iterate is identical (to
many digits) whether we assemble it via the subtraction pattern or
via direct SG addition. The divergence is genuine: the Newton step
direction at that iterate does not admit a sufficient-decrease
length. (This is consistent with the FD-verified Jacobian at the
failing iterate showing `||J - J_fd||/||J|| = 4.24e-9` from PR #45.)

## What this rules in

- The issue is **not** purely positivity (A doesn't help, and the
  1e-30 floor in production is essentially never active).
- The issue is **not** Galerkin-vs-SG cancellation in the assembly
  (C lands on the identical failure).
- The issue **is** Newton step quality at large |dpsi| with full
  carrier redistribution. `bt` correctly rejects the step (it is
  an honest non-descent direction); `cp` accepts it and lands at a
  non-physical fixed point.

## Recommendation

Defer M13.1 1D close-out. The remaining options that are out of
scope for this PR (per the cap on iteration depth and the rule
against modifying FD-verified SG primitives) include:

1. Adaptive time stepping with step-halving on line-search failure.
   The current fixed-dt loop has no recovery; halving dt at step 52
   would likely let the iterate creep through the depletion-region
   redistribution.
2. Pseudo-transient continuation on the first ~100 steps, then
   release to the user dt.
3. Use of the Slotboom (psi, phi_n, phi_p) variables for the
   transient — same as `run_bias_sweep` — which is positivity-free
   by construction. ADR 0009 explicitly chose against this for
   reasons recorded there; revisiting would need a new ADR.
4. Globalisation via `snes_type=ngmres` outside `newtonls`, or a
   damped-Newton wrapper.

Each of these is a separate scope and a separate PR. None should be
attempted under the "follow-up #5" label without an explicit ADR.
