# pmos_idvgs

## What this is

A practical 2D p-channel MOSFET, the natural complement to
[`examples/nmos_idvgs`](../nmos_idvgs). Same channel length (100 nm),
same gate oxide (2 nm SiO2), same Lombardi + Caughey-Thomas + FD
physics path; the only differences are the four sign-flips required
to convert NMOS to PMOS:

1. **Body doping polarity.** NMOS uses a uniform p-type body
   (N_A = 5e17, N_D = 0); PMOS uses a uniform n-type body
   (N_D = 5e17, N_A = 0).
2. **Source/drain doping polarity.** NMOS uses Gaussian donor
   implants (`dopant: "donor"`); PMOS uses Gaussian acceptor
   implants (`dopant: "acceptor"`) with the same peak
   concentration (1e20 cm^-3) and the same Gaussian sigma.
3. **V_DS sign.** NMOS biases the drain positive (V_DS = +0.05 V
   linear, +1.8 V saturation); PMOS biases it negative
   (V_DS = -0.05 V linear, -1.8 V saturation).
4. **V_GS sign.** NMOS sweeps V_GS from 0 V up to +1.8 V; PMOS
   sweeps from 0 V down to -1.8 V (PMOS inversion sits at
   V_GS < V_T_p where V_T_p < 0).

The anchor JSON sweeps V_GS from 0 V to -1.8 V at V_DS = -0.05 V
(linear regime); a companion saturation curve at V_DS = -1.8 V is
generated by the smoke verifier via an in-process re-run that
overrides the drain contact voltage. The intended use case is
"what does an Id-Vgs measurement on a short-channel-ish bulk Si
PMOS look like in the framework," and the config is meant to be
cloned and re-parametrized for your own device.

This is an example, not a V&V gate. For the analytical Pao-Sah I-V
correctness reference at long-channel geometry, see
[`benchmarks/mosfet_2d`](../../benchmarks/mosfet_2d) (NMOS only;
the PMOS analytical reference is the user's exercise via the same
square-law derivation with mobility and band-edge sign flips).

## Physics features exercised

- **M16.1 Caughey-Thomas bulk mobility.** `physics.mobility.bulk_model
  = "caughey_thomas"`. Velocity saturation in the channel under
  drain-side high field (relevant at V_DS = -1.8 V).
- **M16.2 Lombardi surface mobility.** `physics.mobility.model =
  "lombardi"` with `interface_facet_tag = 4` pointing at the gate
  facet. Surface acoustic-phonon and surface-roughness scattering in
  the inversion layer; the dominant ingredient for quantitative MOSFET
  I-V above threshold. The Lombardi formula uses the same hole-mobility
  parameter mu_p as the Caughey-Thomas bulk branch.
- **M16.4 Fermi-Dirac statistics.** `physics.statistics =
  "fermi_dirac"`. The Gaussian source/drain implants peak at
  N_A = 1e20 cm^-3, well above the Boltzmann break point
  (~1e19 cm^-3). The basic Blakemore approximation applies; see
  [`docs/IMPROVEMENT_GUIDE.md`](../../docs/IMPROVEMENT_GUIDE.md) § 4
  M16.4.
- **SRH recombination** with tau_n = tau_p = 10 ns. Auger and
  tunneling are intentionally off here so the run anchors the
  surface-mobility / FD path; they are exercised in
  [`examples/power_diode_reverse_recovery`](../power_diode_reverse_recovery)
  and the M16.6 `zener_1d` benchmark respectively.

## How to run

From the repository root:

```
docker compose run --rm benchmark pmos_idvgs
```

(The CLI in `scripts/run_benchmark.py` falls back to `examples/` when
the requested name is not found under `benchmarks/`, so the same
invocation works for both directories.)

Output lands in `results/pmos_idvgs/`:

- `id_vgs_overlay.png`: linear and semilog |I_D| vs |V_GS| curves
  overlaid for both V_DS values.
- The result JSON written by the engine (with the per-step I-V
  table) is in the standard `results/pmos_idvgs/` location.

## Expected output

At V_GS = -1.8 V, V_DS = -1.8 V the per-width drain current
|I_D| / W is in the rough order-of-magnitude band of the textbook
square-law estimate, scaled down from NMOS by the hole-vs-electron
mobility ratio:

    |I_D| / W = (mu_p,eff * C_ox / (2 * L)) * (V_GS - V_T_p)^2

with mu_p,eff ~ 80-150 cm^2/Vs (Lombardi inversion-layer hole
mobility at 5e17 cm^-3 body), C_ox = eps_SiO2 / t_ox = 1.73e-2 F/m^2
(t_ox = 2 nm), L = 100 nm, |V_T_p| ~ 0.4-0.6 V. Mu_p in Si is roughly
0.3 to 0.4x mu_n at these conditions, so the PMOS saturation current
at the same |V_GS|, |V_DS| should be roughly 30-40 % of the
nmos_idvgs saturation current. The simulation will land within a
factor of about 3 of that estimate; real Lombardi mobility deviates
from the constant assumed by the square-law formula, and velocity
saturation flattens the saturation plateau at high |V_DS|.

I_D itself is **negative** at V_GS = -1.8 V, V_DS = -1.8 V. PMOS
conducts source-to-drain in the negative direction (current flows
from source to drain when V_DS < 0); the verifier asserts this sign
in addition to finiteness.

If the reported |I_D| at V_GS = -1.8 V, V_DS = -1.8 V is more than
roughly 10x off from the square-law estimate, check:

- The interface facet tag (`physics.mobility.interface_facet_tag` must
  match the gate facet tag: 4 in this config).
- The body doping (5e17 cm^-3 here is heavy enough that |V_T_p| moves
  up noticeably; lighter body doping gives a smaller |V_T_p|).
- The S/D Gaussian sigma and the `dopant: "acceptor"` tag (the
  implants must overlap the channel for source-injection of holes to
  work; if the implants are too narrow there is no inversion layer).
- The gate workfunction. The shipped JSON uses workfunction 0
  (mid-gap reference); a real polysilicon gate over n-type body
  shifts V_T_p further negative.

To extract V_T_p from the linear-regime curve (V_DS = -0.05 V), find
the V_GS value at which the linear extrapolation of the steepest-slope
tangent of |I_D| vs |V_GS| crosses zero. The companion C-V analysis
at the same body doping is the user's exercise; the
[`examples/moscap_cv_oxide_thickness`](../moscap_cv_oxide_thickness)
example shows the C-V workflow on a different body type.

## How to adapt

Most users will want to start by changing one of:

- **Channel length.** Edit the mesh extents in x and the Gaussian S/D
  centers proportionally. Same caveats as for nmos_idvgs apply
  (sub-50 nm short-channel effects are captured only qualitatively).
- **Gate oxide thickness.** Edit the y extent (`extents[1][1]`) and
  the oxide region's lower y bound, then re-derive the mesh resolution
  so the oxide remains exactly one cell tall.
- **Body doping.** Change `doping[0].profile.N_D`. Heavier n-type body
  doping shifts |V_T_p| up and flattens the subthreshold slope.
- **S/D doping.** Change the Gaussian peak (`doping[1].profile.peak`
  and `doping[2].profile.peak`). Above 1e20 cm^-3 the Fermi-Dirac
  correction is essential; below 5e18 cm^-3 you can switch back to
  `physics.statistics = "boltzmann"` and the run will be cheaper.
- **Temperature.** `physics.temperature` (default 300 K) is the
  thermal-budget knob.

## Notes on CI runtime

Same parameter-tuning notes as for `nmos_idvgs` apply here. The
shipped configuration uses the 60 x 26 mesh and V_GS step 0.1 V to
keep CI runtime under 5 minutes. The shipped configuration produces
38 SNES solves total (2 V_DS values * 19 V_GS samples each).
