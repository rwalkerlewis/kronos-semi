# nmos_idvgs

## What this is

A practical 2D n-channel MOSFET with a 100 nm gate length, 2 nm SiO2
gate oxide, p-type body (N_A = 5e17 cm^-3), and Gaussian n+
source/drain implants (peak N_D = 1e20 cm^-3). The anchor JSON sweeps
V_GS in [0, 1.8] V at V_DS = 0.05 V (linear regime); a companion
saturation curve at V_DS = 1.8 V is generated by the smoke verifier
via an in-process re-run that overrides the drain contact voltage.
The intended use case is "what does an Id-Vgs measurement on a
short-channel-ish bulk Si NMOS look like in the framework," and the
config is meant to be cloned and re-parametrized for your own
device.

This is an example, not a V&V gate. For the analytical Pao-Sah I-V
correctness reference at long-channel geometry, see
[`benchmarks/mosfet_2d`](../../benchmarks/mosfet_2d). For the
companion C-V analysis, see [`benchmarks/mos_2d`](../../benchmarks/mos_2d).

## Physics features exercised

- **M16.1 Caughey-Thomas bulk mobility.** `physics.mobility.bulk_model
  = "caughey_thomas"`. Velocity saturation in the channel under
  drain-side high field (relevant at V_DS = 1.8 V).
- **M16.2 Lombardi surface mobility.** `physics.mobility.model =
  "lombardi"` with `interface_facet_tag = 4` pointing at the gate
  facet. Surface acoustic-phonon and surface-roughness scattering in
  the inversion layer; the dominant ingredient for quantitative MOSFET
  I-V above threshold.
- **M16.4 Fermi-Dirac statistics.** `physics.statistics =
  "fermi_dirac"`. The Gaussian source/drain implants peak at
  N_D = 1e20 cm^-3, well above the Boltzmann break point
  (~1e19 cm^-3). The basic Blakemore approximation applies; see
  [`docs/IMPROVEMENT_GUIDE.md`](../../docs/IMPROVEMENT_GUIDE.md) § 4
  M16.4.
- **SRH recombination** with tau_n = tau_p = 10 ns. Auger and
  tunneling are intentionally off here so the run anchors the
  surface-mobility / FD path; they are exercised in
  [`examples/power_diode_reverse_recovery`](../power_diode_reverse_recovery)
  and the M16.6 `zener_1d` benchmark respectively.

## How to run

From the repository root:

```
docker compose run --rm benchmark nmos_idvgs
```

(The CLI in `scripts/run_benchmark.py` falls back to `examples/` when
the requested name is not found under `benchmarks/`, so the same
invocation works for both directories.)

Output lands in `results/nmos_idvgs/`:

- `id_vgs_overlay.png`: linear and semilog Id-Vgs curves overlaid for
  both V_DS values.
- The result JSON written by the engine (with the per-step I-V table)
  is in the standard `results/nmos_idvgs/` location.

## Expected output

At V_GS = 1.8 V, V_DS = 1.8 V the per-width drain current
I_D / W is in the rough order-of-magnitude band of the textbook
square-law estimate

    I_D / W = (mu_eff * C_ox / (2 * L)) * (V_GS - V_T)^2

with mu_eff ~ 200-400 cm^2/Vs (Lombardi inversion-layer effective
mobility at 5e17 cm^-3 body), C_ox = eps_SiO2 / t_ox = 1.73e-2 F/m^2
(t_ox = 2 nm), L = 100 nm, V_T ~ 0.4-0.6 V (depends on the body
acceptor concentration and the metal/Si workfunction offset). For
those numbers the square-law estimate is ~10^2 to 10^3 A/m.

The simulation will land within a factor of about 3 of that estimate;
real Lombardi mobility deviates from the constant assumed by the
square-law formula, and velocity saturation flattens the saturation
plateau at high V_DS.

If the reported I_D at V_GS = 1.8 V, V_DS = 1.8 V is more than
roughly 10x off from the square-law estimate, check:

- The interface facet tag (`physics.mobility.interface_facet_tag` must
  match the gate facet tag: 4 in this config).
- The body doping (5e17 cm^-3 here is heavy enough that V_T moves up
  noticeably; lighter doping gives a lower V_T).
- The S/D Gaussian sigma (the implants must overlap the channel for
  source-injection to work; if the implants are too narrow there is
  no inversion layer to inject into).

To extract V_T from the linear-regime curve (V_DS = 0.05 V), find the
V_GS value at which the linear extrapolation of the steepest-slope
tangent crosses zero current. The companion C-V geometry in
[`benchmarks/mos_2d`](../../benchmarks/mos_2d) gives the V_T from a
quasi-static gate sweep and is a useful cross-check.

## How to adapt

Most users will want to start by changing one of:

- **Channel length.** Edit the mesh extents in x and the Gaussian S/D
  centers proportionally. Shrinking the channel (sub-50 nm) introduces
  short-channel effects (DIBL, V_T roll-off) that the present
  Slotboom kernel captures only qualitatively; for production short-
  channel work, expect to bump the body doping and refine the mesh.
- **Gate oxide thickness.** Edit the y extent (`extents[1][1]`) and
  the oxide region's lower y bound, then re-derive the mesh resolution
  so the oxide remains exactly one cell tall (the engine tags cells by
  centroid; if the oxide region's vertical extent contains no cell
  midpoint, the oxide effectively disappears).
- **Body doping.** Change `doping[0].profile.N_A`. Heavier body doping
  raises V_T and flattens the subthreshold slope.
- **S/D doping.** Change the Gaussian peak. Above 1e20 cm^-3 the
  Fermi-Dirac correction is essential; below 5e18 cm^-3 you can switch
  back to `physics.statistics = "boltzmann"` and the run will be
  meaningfully cheaper.
- **Temperature.** `physics.temperature` (default 300 K) is the
  thermal-budget knob. Above ~400 K you may want to add the Arora
  doping-dependent mobility branch (not yet shipped); below ~150 K the
  Boltzmann approximation breaks even more thoroughly.

## Notes on CI runtime

The prompt's nominal sweep was V_DS in [0.05, 0.5, 1.0, 1.8] V at
V_GS step 0.05 V on a 200 x 80 mesh. Two parameter knobs were tuned
to keep CI runtime under 5 minutes without setting `allow-failure:
"true"`:

- V_DS sample count: dropped from four to two (linear-regime anchor at
  V_DS = 0.05 V plus the saturation-regime companion at V_DS = 1.8 V).
- V_GS step: 0.1 V (the prompt's first fallback knob).
- Mesh: 60 x 26 cells (1560 cells; the prompt's 200 x 80 mesh is a
  16k cell budget which the SNES does not absorb cheaply at FD
  statistics with N_D = 1e20 cm^-3 implants).

The shipped configuration produces 38 SNES solves total (2 V_DS *
19 V_GS samples) with the FD plus Lombardi plus Caughey-Thomas
physics path active.
